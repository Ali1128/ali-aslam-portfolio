Case study 02

Asset Status Cross-Check Tool

A tool concept for comparing expected deliverables against received media assets across video, audio, subtitles, captions, artwork, metadata, regions, and language versions.

Expected vs received Fictional dataset QA support
Role focus
Validation logic and interface concept
Primary users
Media operators, coordinators, and QC teams
Output
Delivery gap summary and exception list

The Problem

Delivery packages can include many expected components. When operators manually compare received files against title requirements, issues like missing language versions, duplicate files, naming inconsistencies, and unexpected versions can slip through or take too long to verify.

My Approach

The concept creates a simple comparison layer: load the expected deliverables, load the received inventory, normalize naming and language codes, then generate an exception report that is easy to review and escalate.

Process or Workflow

  1. Define the expected package by title, territory, asset type, and language.
  2. Import a received file list from a fictional delivery folder or CSV export.
  3. Normalize common naming patterns and language code variants.
  4. Flag missing assets, duplicate files, unexpected versions, and delivery gaps.
  5. Export a human-readable summary for follow-up.

Tools and Technologies

  • CSV or JSON inputs for expected and received files
  • JavaScript or Python for rule-based comparison
  • Configurable language and territory mappings
  • Spreadsheet-friendly exception exports

Results or Expected Impact

The tool could reduce repetitive manual comparison work, make delivery gaps visible earlier, and give teams cleaner language for vendor follow-up. It would not replace human review, but it would help operators prioritize the exceptions that need attention.

Key Lessons

Reliable comparison depends on clean expectations. Before automation is useful, teams need consistent required asset profiles, agreed language mappings, and clear exception categories.

Next Steps

Next improvements could include drag-and-drop CSV upload, configurable naming patterns, severity rules, and a printable delivery gap report.
